# Linux / Nginx 점검 명령어 체크리스트

## `systemctl status nginx`

- 목적: Nginx 서비스 실행 상태 확인
- 정상 기준: `active (running)`
- 장애 기준: `inactive`, `failed`

## `sudo nginx -t`

- 목적: Nginx 설정 파일 문법 검사
- 정상 기준: `syntax is ok`, `test is successful`
- 실무 의미: 설정 반영 전 오류를 사전에 확인

## `ss -lntp`

- 목적: 서버가 어떤 TCP 포트에서 요청을 기다리는지 확인
- 정상 기준: Nginx가 `0.0.0.0:80` 또는 `[::]:80`에서 Listen

# Linux / Git 실무 파일 및 로그 제어 체크리스트

## `ls -l /var/www/html`

- 목적: 웹 루트 디렉토리 내부 파일 목록 및 상세 권한 확인
- 정상 기준: `index.html` 또는 기본 파일이 존재하며, 권한이 `-rw-r--r-- (644)`로 지정됨
- 실패 현상: `total 0` (텅 비어있음) 또는 권한이 `000` 등으로 막혀 웹서버 접근 불가
- 실무 의미: Nginx가 파일을 읽을 수 있는 정상 권한(외부 읽기 가능)인지 확인

## `sudo cp [원본파일] [백업파일]`

- 목적: 환경 설정이나 콘텐츠 수정 전 원본 파일 안전하게 백업
- 정상 기준: 에러 메시지 없이 조용히 명령어가 실행됨 (`ls -l` 시 `.bak` 파일 생성 확인)
- 실패 현상: `missing destination file operand` (사본 경로를 누락했을 때 발생)
- 실무 의미: 운영 서버 환경을 건드리기 전 안전장치

## `sudo nano /var/www/html/index.html`

- 목적: GUI가 없는 환경에서 터미널 내부 내장 에디터로 파일 수정
- 정상 기준: 에러 없이 텍스트 편집 화면이 열림
- 주의 사항: 경로에 오타가 날 경우 기존 파일이 열리지 않고 텅 빈 '유령 새 파일'이 생성됨
- 탈출 단축키: 변경 저장(`Ctrl + O` -> `Enter`) 후 종료(`Ctrl + X`)

## `tail -n 20 /var/log/nginx/access.log`

- 목적: Nginx 웹서버의 최신 접속 기록(로그) 20줄 확인
- 정상 기준: `curl` 요청 시 맨 밑바닥에 실시간으로 `HTTP 200` 및 접속 정보 데이터가 한 줄씩 적재됨
- 실무 의미: 과거 기록(`head`)이 아닌 가장 최근의 현재 장애 상황이나 트래픽 유입을 실시간으로 추적

## `git commit -am "메시지"`

- 목적: 수정된 기존 파일들을 대기실 등록(`add`)과 최종 저장(`commit`) 한 방에 처리
- 정상 기준: `[main xxxxxx] 메시지` 문구와 함께 변경된 파일 수와 줄 수가 출력됨
- 실패 현상: 신규 파일(Untracked file)이 있을 때는 이 단축키가 먹히지 않고 거부됨
- 실무 의미: 기등록된 파일들의 미세한 오타 수정이나 문서 업데이트 시 사용
