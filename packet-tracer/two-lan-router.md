# Packet Tracer Two-LAN Routing Lab

## 목적

서로 다른 두 LAN을 Router로 연결하고,
Default Gateway, ARP, ICMP, Routing 흐름을
Simulation Mode에서 확인한다.

## Topology

```text
PC-A
  ↓
SW-A
  ↓
R1
  ↓
SW-B
  ↓
PC-B
```

## 내가 이해한 내용

- 패킷트레이서를 이용해 기초적인 PC1-SW1-R1-SW2-PC2를 구성해서 IP, Gateway, Subnetmask를 할당한 뒤에 PING이 정상적으로 도달하는 지 확인했다.
- PC1에서 다른 네트워크에 있는 PC2로 PING을 보내면 바로 PC2로 도달하는 것이 아니라 먼저 다른 네트워크 대역에 있기 때문에 PC1(192.168.10.10)의 Gateway(192.168.10.1)의 MAC을 찾아서 기억한 뒤(ARP) 실제로 데이터가 발송되고 라우터가 다른 네트워크의 경로를 찾은 후, 라우터가 PC2의 MAC 주소를 실제로 찾은 뒤, 최종적으로 PC2에 도달하고 되돌아올 때도 같은 방법을 쓴다.
- 시뮬레이터 MODE로 봉투가 흘러가는 되돌아오는 과정을 지켜봤다.

## 헷갈린 내용

- PING 192.168.20.10으로 컴퓨터에서 명령어를 내렸을 때는 논리적인 부분만 이해하고 있었지 실제로 어떻게 움직이는 지 정확히는 알지 못했지만 실제로 패킷트레이서를 통해 흐름을 보니까 이해할 수 있었다.
- Gateway Mac 주소를 찾을 때 라우터까지 한 번 요청을 했다가 되돌아오고 MAC 주소를 기억하면 그 때 실제로 신호가 가는지 헷갈린다.

## 한 문장 요약
- 다른 네트워크로 요청을 보내면 바로 보내는 것이 아니라 먼저 설정된 게이트웨이 MAC 주소를 ARP로 확인 후 패킷을 라우터에 전달한 후, 라우터는 목적지의 IP 주소를 기준으로 경로를 설정한 뒤 요청을 받을 PC의 MAC주소를 ARP로 확인해서 최종 전달한다.
