# AWS VPC Public Subnet Basic Lab

## 목적

AWS에서 VPC와 Public Subnet을 생성하고,
Internet Gateway와 Route Table을 연결하여
인터넷 연결이 가능한 Network 구조를 구성한다.

---

## 전체 구조

```text
Internet
   │
Internet Gateway
bj-lab-igw
   │
Route Table
10.10.0.0/16 → local
0.0.0.0/0 → IGW
   │
Public Subnet
10.10.1.0/24
   │
EC2 예정
```

## 내가 이해한 내용

- AWS VPC는 가상 네트워크 공간을 의미한다고 이해했다.
- VPC를 하나 만들고 10.10.0.0/16 범위 안에서 10.10.1.0/24로 서브넷 하나를 생성했다.
- 외부 네트워크 통신을 위한 출구를 인터넷 게이트웨이라고 이해했으며, Attach로 내가 만든 VPC와 연결했다.
- 여기서 끝이 아니라 라우팅 테이블이라는 '경로 장부'를 만들어서 0.0.0.0/0 이라는 외부 네트워크 통신을 위한 라우팅, 10.10.0.0/16이라는 내부 라우팅을 확인했다.
- 마지막에는 인바운드 규칙에서 내 IP에서 들어오는 트래픽을 SSH 22번 포트, 인터넷에서 들어오는 모든 트래픽은 HTTP 80번 포트로 설정했고 향후 실습을 위한 원격 접속과 Nginx 통신을 위해서 열어둔 것이라고 이해했다.
- VPC - 서브넷 - 인터넷 게이트웨이(VPC와 연결) - 라우팅 테이블(외부 및 내부 경로설정) 이라는 흐름을 이해했다.

## 오늘 헷갈린 점

- 10.10.0.0/16이라는 CIDR 표기법이 처음으로 나와서 당황했다.
- 나는 /16이니까 약 6만 5천개의 호스트만 사용 가능한 것인가 하고 의문을 가졌지만 부모 서브넷을 다시 /24로 나눠서 여러 서브넷을 만들 수 있는 것이 조금 헷갈렸다.
- 10.10.10.0/24가 아니라 왜 10.10.1.0/24를 쓰는 지 헷갈렸다
- 아직은 처음이라 VPC 생성부터 보안그룹까지 중간중간에 생성하고 연결하고 하는 부분이 많아서 어려웠다.
